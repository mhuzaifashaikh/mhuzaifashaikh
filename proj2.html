In-Vehicle Information System-
Data Management .

A quick demonstration going over the main functionality of the Program.

I was tasked with creating a program for In-Vehicle Information Systems and this program is what I came up with. My program was required to take in user entires, check if it already exists, output an error if it does and ask user to enter again, add to database if it did not already exist and assign a value to each entry, allow the user to to search for an entry, allow the user to delete an entry and so on... Below I go into detail on how I decided to desgin this program.


How I designed this project.

In this Project, I chose to implement this algorithm primarily using hashmaps. I chose to use hashmaps as they are the most efficient and easiest way to store and deal with entries that contain unique keys for each value. Due to the primary system performance criteria being speed, I utilised hash maps for quick search of objects using a key. Hashmaps also was the best choice as speed was a criteria but the order of the key-value pairs being maintained was not important.
In this Project I have also used an array list as a supplementary data structure that helps in storing the keys. The index at which a specific key is stored is used as the value of that key. In that way no two keys will have the same value either(Since two items cannot be stored at the same index!).


Methods/Functions

Function 1- Insert

The insert function is used to essentially add the key-value pair into the hashmap. This is done by first checking if the key already exists in the hashmap, because if it does we will have to alert the user and return from the method as each key is supposed to be unique.
However, if the key does not exist, we proceed to add it to the hashmap. This is done by adding the key to the supplementary array list. Once it is added to the array list, the index of the key is used as the value for this key and this pair is added to the hashmap.


Function 2- Delete

The delete function is used to essentially delete the key-value pair from the hashmap. This is done by first checking if the exists in the hashmap. That is done by extracting the value of the key using a hashmap method and checking if the value is null. If it is, then the key does not exist and therefore there is nothing to delete and the user is alerted subsequently.
However, if the key does exist, we proceed to delete it from the hashmap. This process is a little more complicated than inserting. It would be easier to remove an element from the end of an array list rather than the middle which would need it to then rearrange all the rest of the elements so that no value in between the array list is null. And since the order of the keys is not important, what we did is swap the key to remove with the last key in the arraylist. We did this buy extracting the index of the last element of the array list (size of arraylist -1) and then checked to see if this index is also the same as the value of the key, because if it is that would mean that this key-value pair is the only one in the arraylist(and therefore in the hashmap) and we would not need to do any swapping. However, if it is not, then we extract the current key in the last position of the array list to later assign a new value to it. After we do that, we use a java utility function to swap in the array the key in the last position with the key that we have to delete. And once we do that, we remove the key that is now in the last position(this is now the key we want to delete).
Finally, assign the key that was swapped the new value which was previously used byt the key we just removed.


Function 3- Search

The search function is used to essentially search a key-value pair on the hashmap. This is done by first checking if the exists in the hashmap. That is done by extracting the value of the key using a hashmap method and checking if the value is null. If it is, then the key does not exist and therefore it is not found and the user is alerted subsequently.
However, if the key does exist, then we print a message to the user that shows the key they searched for and the value of this key. The function then returns this value.
